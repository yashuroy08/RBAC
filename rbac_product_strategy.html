RBAC Risk Evaluator — Product strategy overview including project aim, feature roadmap, and visual identity

Project aim

Secure, adaptive access control — with real-time intelligence

This system solves the gap between static role-based permissions and dynamic, context-aware security. It gives organisations a single platform to enforce who can access what, detect suspicious session behaviour automatically, and respond to risk — without manual intervention. The target users are IT administrators managing multi-user environments where concurrent sessions, device trust, and physical location are all security variables.

Feature roadmap

01

Real-time threat dashboard

High impact

A live admin view showing risk scores, active sessions, and flagged events across all users simultaneously — with WebSocket push updates, not polling. Replaces the current per-user query model.

02

Behavioural anomaly detection

High impact

Move beyond session count. Detect unusual login hours, rapid IP hops, and impossible travel between sessions. Build a rolling baseline per user and auto-escalate MFA when deviation is detected.

03

Self-service user security portal

Medium priority

Let users view, name, and revoke their own trusted devices and sessions without contacting an admin. Reduces support burden and increases transparency — users can see exactly what has access to their account.

04

Audit log and compliance export

Medium priority

Immutable, filterable logs of every risk event, login, role change, and session invalidation. Export to CSV or PDF for GDPR, ISO 27001, or SOC 2 compliance reporting. Includes admin action attribution.

05

Notification and alert system

Quick win

Email or webhook alerts when thresholds are breached, a new device is trusted, or an account is locked. Configurable per-admin. Currently the system acts silently — users only discover events on next login.

Visual identity

Precision Dark
Command Light

Aesthetic direction: Structured Utility — the visual language of professional security tooling. Think enterprise-grade clarity: high information density, strong typographic hierarchy, and purposeful use of colour only to encode meaning (risk level, status, role). No decorative elements. Inspired by tools like Datadog, Okta, and Linear.

Key principles: monospaced session IDs and device hashes for legibility, traffic-light status colours (red/amber/green) used consistently and never decoratively, and a persistent risk score gauge visible on every authenticated screen.

Colour usage map — what each colour encodes

Blue ramp

Primary actions, navigation, ADMIN role badges, session IDs, interactive elements

Green ramp

Risk score LOW, trusted device confirmed, account unlocked, session within limit

Amber ramp

Risk score MEDIUM, approaching session limit, MFA prompted, warning states

Red ramp

Risk CRITICAL, session invalidated, account locked, threshold breached alerts

Purple ramp

MFA states — pending verification, device trust level, OTP flow indicators

Gray ramp

Inactive sessions, disabled accounts, historical events, secondary metadata
